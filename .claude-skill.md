# Knowledge Wiki Skill

## 协作规范 (v2.0 优化版)

### 1. 文档结构规范
所有 Markdown 文档必须遵循以下顺序：

```markdown
---
title: 文档标题
date: YYYY-MM-DD
tags: [tag1, tag2]
---

# 文档标题

> **摘要**: 这里写一句话总结或精简的文字总结，直接说明做了什么、解决了什么问题。

## 背景与需求
...

## 技术方案
...
```

**注意**：
- **元信息 (YAML Frontmatter)**：必须保留在文件最顶部（为了 VitePress 正确解析），但在正文中不再显式展示其代码块。
- **摘要区**: 紧跟标题，使用引用块 `> ` 格式，快速传递核心价值。

### 2. 变更日志 (Changelog) 规范
在 `docs/changelog.md` 中记录时，采用以下简洁格式：

```markdown
### 文档标题
- **类型**: iteration | script | feature
- **摘要**: 一句话描述
- **文档**: [链接](./path/to/doc.md)
- **状态**: 🟢 Active | 🔵 Completed
- **Last Modified**: YYYY-MM-DD HH:mm
```
- **标题**: 不再包含时间戳。
- **时间**: 统一使用 `Last Modified` 字段。

### 3. 创建新文档流程
0. **永远使用中文**
1. **复制模板**: 从 `docs/templates/` 选择对应类型的模板。
2. **填写元信息**: 仅在文件顶部保留 YAML 块。
3. **编写摘要**: 在 H1 标题下立即添加摘要。
4. **更新日志**: 按新格式更新 `docs/changelog.md`。
5. **本地验证**: `npm run build`。

### 4. 发布前置检查
1. **本地构建**: 运行 `npm run build`。
2. **死链检查**: 确保无 `(!) Found dead link` 警告。
3. **Skill 同步**: 运行 `npm run sync-skills`。
