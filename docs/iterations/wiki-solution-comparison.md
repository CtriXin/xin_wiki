## 背景

在建立个人知识库之前，对市面上主流的文档管理方案进行了调研，以确定最适合长期维护和高效阅读的技术栈。

## 方案对比

### 1. 纯 Markdown + GitHub
- **优点**: 零成本、版本控制、随时编辑、专注内容
- **缺点**: 阅读体验一般、无搜索、无分类浏览
- **适合**: 个人快速记录

### 2. VitePress 网站 ⭐ 最终选定
- **优点**:
    - 🎨 现代化设计，具备极佳的视觉效果
    - 🔍 内置全文搜索（本地索引）
    - 🌙 自动暗黑模式切换
    - 📱 完美适配移动端
    - ⚡ 静态生成，极速加载
    - 📝 保持 Markdown 原生编辑体验
- **缺点**: 需要 Node.js 环境构建
- **适合**: 对外展示、长期维护、追求极致阅读体验

### 3. Notion / 语雀
- **优点**: 可视化编辑、实时协作、功能丰富
- **缺点**: 依赖第三方平台、数据导出困难、自定义受限
- **适合**: 团队协作

### 4. 自建专业 Wiki (MediaWiki)
- **优点**: 功能最强大、专业百科体验
- **缺点**: 维护成本高、需要服务器、配置极其复杂
- **适合**: 大型开源项目文档

## 决策建议

**结论**: VitePress 是个人技术知识库的最佳平衡点。
- 它允许我们保留纯粹的 Markdown 内容（保证了数据的持久性和可迁移性）。
- 同时提供了一个现代、美观且功能完备（搜索、导航）的展示层。
