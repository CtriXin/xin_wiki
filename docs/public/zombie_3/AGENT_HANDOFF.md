# Zombie 项目交接文档（给新 Session / 新 Agent）

## 1. 项目定位
- 路径：`/Users/xin/knowledge-wiki-web/docs/public/zombie`
- 形态：纯前端 Canvas 游戏（无后端）
- 主题：孩子手绘启发的火柴人防守 + 职业 + 武器铸造 + 波次

## 2. 当前主要功能
- 4 职业：战士 / 弓箭手 / 法师 / 刺客
- 每职业 4 武器：
  - 默认仅第 1 把解锁
  - 其余在装备台“铸造下一把武器”解锁
  - 最强武器有使用限制（冷却 / 连击 / 目标类型）
- 技能 3 阶成长：随波次提升（T1/T2/T3）
- 基地防守：怪接触基地会扣血，基地可修复
- 波次系统：前两波降难度（怪更少、更慢）
- 波次系统：每波进入“备战阶段”，玩家点击开始后才出怪（非自动衔接）
- 备战面板含“打开合成台”快速入口，避免找不到合成按钮
- 每波“开始战斗”按钮独立悬浮，不依赖备战面板是否关闭
- 每波开战后左右信息栏自动折叠成箭头，点击箭头可展开
- 右侧新增“画风”按钮：线稿风/方块风一键切换
- 右侧新增“暂停”按钮：暂停时停止战斗更新，仅保留渲染
- 波间战术抉择：每波结束出现 3 选 1，超时自动选第一个
- 新技能机制（按强度成长）：
  - 战士：嘲讽（怪转向玩家，不打房子）
  - 刺客：范围致盲（2/4/6/8/10格）
  - 法师：范围全体冰冻
  - 弓箭手：万箭齐发（区域持续掉血）
- 手绘自定义：人物/怪物/房子/弹道/掉落图标/背景印记
- 引导系统：4 步引导后才开局
- 紧凑模式：引导完成后按钮图标化，减少遮挡

## 3. 关键文件
- `index.html`：DOM 结构（按钮、面板、引导、战术抉择）
- `styles.css`：UI 样式与紧凑模式
- `src/main.js`：DOM 事件绑定、引导流程、面板交互
- `src/game.js`：核心状态、主循环、渲染、战斗、掉落、战术逻辑
- `src/game.js`：核心状态、主循环、渲染、战斗、掉落、战术逻辑、技能与合成配方
- `src/config.js`：职业、稀有度、层级、武器与技能成长配置
- `PROJECT_SPEC.md`：项目设计说明（持续更新）

## 4. 操作说明（给开发者）
- 运行方式：直接静态打开 `index.html` 或通过本地静态服务器
- 语法检查：
  - `node --check src/main.js`
  - `node --check src/game.js`
  - `node --check src/config.js`

## 5. 已知设计决策
- `classBar` 负责“职业选择”
- `inventory` 负责“当前职业的武器槽（4把）”
- 武器推荐已改为右侧折叠抽屉（默认可收起）
- 左右信息区为展示层，默认点击穿透

## 6. 推荐后续迭代
- 合成台做成 2x2 工作台格子（更像孩子画里的 MC 风格）
- 2x2 合成网格已接入：点击格子切换材料，按材料比例触发概率结果
- 合成支持失败率、同比例多结果、重复配方熟练度降低失败率
- “开始合成”是独立执行按钮；上方有当前配方预览
- “开始合成”带确认进度动效（约0.56s），完成后才真正执行合成
- “自定义银武”改为隐藏触发：需有自定义画稿 + 触发秘密配方
- 掉落扩展：除💎/❤️/🔥外，新增🧩与🔷掉落
- 引入怪物词条图标（装甲/迅捷/群体/进化）的小图例
- 增加“暂停/继续”按钮（给孩子中断与讲解）
- 做一套低龄模式（更大按钮、更慢节奏、更少文本）

## 7. 回归测试清单
- 开局必须完成 4 步引导后才开战
- 武器未铸造时不能装备，提示正确
- 最强武器限制触发与提示正确
- 波间战术面板出现、可点选、超时自动选
- 右侧“武器推荐”可展开/收起，不遮挡主战场
- 自定义保存后生效，恢复默认可还原
- 战士冲锋与刺客背刺手感明显不同
